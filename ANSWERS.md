 1. How to run
Open index.html directly in a browser. No installation required.

 2. Stack & design choices
I used vanilla HTML, CSS, and JavaScript because the project is small and does not require frameworks.

I designed the tip buttons as a grouped selector so only one option can be active at a time, making it behave like a radio control.

I placed output values in a separate panel so updates feel real-time and visually separated from inputs.

 3. Responsive & accessibility
On mobile (360px), inputs stack vertically and buttons remain full width for easier tapping. On larger screens (1440px), the layout remains centered for readability.

I included basic labels for accessibility. I did not fully implement ARIA attributes because the project scope was small and already readable with native HTML semantics.

 4. AI usage
I used ChatGPT to help structure the calculation logic and event handling flow. I reviewed and adjusted the code to match my own HTML structure and simplified some parts for readability, such as replacing complex selectors with direct IDs.

 5. Honest gap
The UI styling is basic and not fully polished. With more time, I would improve spacing, typography, and make the layout more responsive for different screen sizes.
